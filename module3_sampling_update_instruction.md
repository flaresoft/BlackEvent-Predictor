# Module 3 샘플링 전략 변경 요청

## 변경 사항 요약
- 샘플 수: 1,000건 → **5,000건**
- 샘플링 방식: 단순 랜덤 → **층화 샘플링 (Stratified Sampling)**

## 배경
- 2006~2026년 20년간의 뉴스 데이터를 사용
- 최근 연도로 갈수록 기사 수가 많아서, 단순 랜덤 추출 시 최근 기사에 편중됨
- 시대별 문체, 주제, 표현이 다르므로 각 시기의 성질을 골고루 발견해야 함
- 수확체감 분석 결과 5,000건이 비용 대비 성질 발견 효율의 최적점

## 층화 샘플링 설계

### 축 1: 시기별 균등
데이터 기간을 5개 구간으로 나누어 각 구간에서 동일 수량 추출:
- 구간 A: 2006~2009
- 구간 B: 2010~2013
- 구간 C: 2014~2017
- 구간 D: 2018~2021
- 구간 E: 2022~현재

### 축 2: BlackEvent 직전 vs 평상시 균등
각 시기 구간 내에서 다시 절반씩 배분:
- 50%: BlackEvent first_shock_date 직전 60일 이내 기사
- 50%: 대조군(평상시) 기간 기사

### 최종 배분표 (5,000건)
| 구간 | BlackEvent 직전 | 평상시 | 소계 |
|------|----------------|--------|------|
| 2006~2009 | 500건 | 500건 | 1,000건 |
| 2010~2013 | 500건 | 500건 | 1,000건 |
| 2014~2017 | 500건 | 500건 | 1,000건 |
| 2018~2021 | 500건 | 500건 | 1,000건 |
| 2022~현재 | 500건 | 500건 | 1,000건 |
| **합계** | **2,500건** | **2,500건** | **5,000건** |

### 예외 처리
- 특정 구간에서 해당 유형 기사가 500건 미만이면, 해당 구간에서 가능한 만큼 추출하고 부족분은 인접 구간에서 보충
- 부족분 보충 시에도 BlackEvent/평상시 비율 50:50은 유지

## 수정 대상

### 1. `BlackEvent_Predictor_Architecture.md`의 Module 3 섹션
**Step 3-1: 샘플 오픈형 태깅** 부분을 아래와 같이 변경:

기존:
```
- 전체 코퍼스에서 1,000건 무작위 샘플링
- 샘플링 시 BlackEvent 직전 / 평상시 균등 배분
```

변경:
```
- 전체 코퍼스에서 5,000건 층화 샘플링(Stratified Sampling)
- 층화 기준 1: 시기별 균등 (2006~2009 / 2010~2013 / 2014~2017 / 2018~2021 / 2022~현재, 각 1,000건)
- 층화 기준 2: 각 시기 내 BlackEvent 직전 / 평상시 균등 배분 (각 500건)
- 특정 구간에서 해당 유형 기사가 부족하면 인접 구간에서 보충 (비율 유지)
```

**Step 3-1의 출력 부분도 변경:**
기존: `총 3,000~5,000개 raw 태그`
변경: `총 15,000~25,000개 raw 태그`
